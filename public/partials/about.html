HookRel

Webhooks in. Notifications out.

HookRel is a self-hosted webhook relay system. It receives webhook events from your applications, matches them against your rules, formats them using templates, and dispatches notifications to your chosen providers — all without writing a single line of code.

📡

Sources

Each source represents an application that sends webhooks — Sonarr, Radarr, Gitea, or any custom app. Every source gets a unique URL and bearer token for authentication.

🔔

Providers

Providers are where notifications go. HookRel supports SMTP email, Microsoft Graph (Microsoft 365), Telegram bots, and Pushover push notifications.

📝

Message Templates

Message Templates define the notification format using {{token.path}} syntax. Use the built-in rich editor to build HTML emails with formatting, tables, and colour — or plain text for plain-text providers.

🔀

Rules

Rules wire everything together — matching a source and event type to a provider and template. Multiple rules can fire for the same event, or just the first match depending on your match mode setting.
