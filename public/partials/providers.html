Telegram Settings
Bot Token — open Telegram and message @BotFather. Send /newbot, follow the prompts, and copy the token it gives you (format: 123456789:ABCdef...).
Chat ID — add your bot to a chat or channel, then message @userinfobot in that chat to get its ID. Channel IDs start with -100.
Pushover Settings
API Token — log in at pushover.net, go to Your Applications and create a new application. The API token is shown on the application page.
User Key — found on your Pushover dashboard at pushover.net under Your User Key (top right of the main page, starts with u).
Discord Settings
Webhook URL — in Discord, open the channel settings → Integrations → Webhooks → New Webhook. Copy the webhook URL (format: https://discord.com/api/webhooks/ID/TOKEN).
Slack Settings
Webhook URL — go to api.slack.com/apps, create an app, enable Incoming Webhooks, and add a webhook to your workspace. Copy the URL (format: https://hooks.slack.com/services/...).
Gotify Settings
Server URL — the base URL of your Gotify instance (e.g. https://gotify.example.com).
App Token — in Gotify go to Apps, create a new application, and copy its token.
ntfy Settings
Server URL — use https://ntfy.sh for the public server, or your self-hosted instance URL.
Topic — a unique name for your notification channel (e.g. hookrel-alerts). Anyone who knows the topic can subscribe, so use something unguessable or enable auth.
Token — optional. Required if your ntfy server has access control enabled. Generate via ntfy token add or the web UI.
Microsoft Teams Settings
Webhook URL — in Teams, go to the channel where you want notifications → click ··· → Workflows → search for "Post to a channel when a webhook request is received" → follow the setup and copy the URL provided at the end.
